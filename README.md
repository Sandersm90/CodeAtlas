# CodeAtlas

A Claude Code plugin that gives every project a persistent, searchable wiki knowledge base. Semantic search via Ollama, keyword search via BM25, and full lifecycle management ‚Äî all accessible as MCP tools and slash commands.

---

## What it does

### Ingest
- **`wiki_ingest`** ‚Äî process raw sources into wiki pages via Claude
  - `file` / `files[]` ‚Äî local files from `RAW_ROOT`
  - `url` / `urls[]` ‚Äî fetch from web pages or GitHub file URLs directly (GitHub blob URLs auto-converted to raw)
  - Deduplication: embeds a sample of each source and surfaces similar existing pages before Claude writes anything
- **`/wiki-ingest <file>`** ‚Äî ingest shortcut

### Search & retrieval
- **`wiki_search`** ‚Äî hybrid semantic + keyword search; supports `tags` filter
- **`wiki_get`** ‚Äî fetch a full wiki page by name
- **`wiki_list`** ‚Äî enumerate all pages with title, tags, updated date; supports tag filtering
- **`wiki_context_for`** ‚Äî given a source file path, extract filename + symbols and return the most relevant wiki pages automatically
- **`/wiki-search <query>`** ‚Äî search shortcut

### Write
- **`wiki_update`** ‚Äî create or update a page and re-embed it
  - `dry_run: true` ‚Äî preview changes without writing
  - `git_commit: true` ‚Äî `git add + commit` after writing
  - Incremental re-embedding: unchanged chunks skip Ollama, only changed chunks re-embed
- **`wiki_delete`** ‚Äî remove a page and its vectors
- **`wiki_rename`** ‚Äî rename a page and rewrite all incoming `[[links]]` atomically

### Maintenance
- **`wiki_lint`** ‚Äî health check: broken links (with fuzzy suggestions), orphan pages, missing frontmatter, stale embeddings, missing concepts
  - `fix: true` ‚Äî auto-fix missing `updated` dates in frontmatter
- **`wiki_reembed_all`** ‚Äî batch re-embed stale pages (default) or all pages; returns `{reembedded, skipped, errors, total}`
- **`/wiki-lint`** ‚Äî lint shortcut

### Bootstrap
- **`/wiki-init`** ‚Äî bootstrap the wiki in any project (creates `wiki/`, `raw/`, `.mcp.json`)

The wiki skill auto-triggers: Claude searches before starting a task, updates after implementing something, and ingests when you provide a spec or notes.

## Requirements

- [Claude Code](https://claude.ai/code)
- [Ollama](https://ollama.com) running somewhere accessible (local or homelab)
- `nomic-embed-text` model pulled in Ollama: `ollama pull nomic-embed-text`
- Node.js 18+

## Installation

```bash
# 1. Install the CLI globally (builds native modules correctly)
npm install -g @synnode/codeatlas

# 2. Add the CodeAtlas marketplace
/plugin marketplace add synnode/codeatlas

# 3. Install the wiki plugin
/plugin install wiki@codeatlas
```

Then in any project you want to use the wiki:

```
/wiki-init
```

This creates `wiki/`, `raw/`, `.mcp.json`, and updates `CLAUDE.md`. Edit `.mcp.json` to set your Ollama URL, then restart Claude Code.

## Wiki page format

Every page is a markdown file with YAML frontmatter:

```markdown
---
title: "AuthMiddleware"
tags: [architecture, auth]
related: ["User", "Session"]
updated: 2026-05-02
---

# AuthMiddleware

One-paragraph summary.

## Overview
...

## Key Decisions
...

## Related
- [[User]]
- [[Session]]
```

Required frontmatter fields: `title`, `tags`, `updated`. Use `[[PageName]]` for cross-references.

## Configuration

Set per-project in `.mcp.json` (generated by `/wiki-init`):

| Variable | Description | Default |
|---|---|---|
| `WIKI_ROOT` | Path to project's `wiki/` folder | ‚Äî |
| `RAW_ROOT` | Path to project's `raw/` folder | ‚Äî |
| `OLLAMA_URL` | Ollama base URL | ‚Äî |
| `OLLAMA_MODEL` | Embedding model | `nomic-embed-text` |
| `DB_PATH` | sqlite-vec database path | `{WIKI_ROOT}/.embeddings.db` |
